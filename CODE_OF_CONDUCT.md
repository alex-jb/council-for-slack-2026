# Code of Conduct

Council for Slack is a calibration tool. It only works if the people using and building it are honest about what they don't know.

## Three rules

1. **Disagreement is the signal.** This project exists because the council pattern exposes disagreement that single-LLM tools erase. Treat disagreement the same way in code review, issue threads, and PR discussion. "I think you're wrong because..." is welcome. "I think you're stupid because..." is not.

2. **Scored claims only.** If you say "this is faster", show the numbers. If you say "this is more correct", show the test that fails today and passes after. If you say "users want this", point at the support ticket or the survey. We are building a Brier-audited tool — bring the same discipline to how we work on it.

3. **Be kind to the person; ruthless on the argument.** A bad PR is fixable. A pattern of contempt is not. Critique the code, not the contributor. Critique the argument, not the arguer's perceived intelligence, background, or motive.

## What we will not tolerate

- Personal attacks, harassment, doxing, or threats — in issues, PRs, commits, or anywhere else this project touches.
- Discrimination on the basis of race, gender, sexual orientation, national origin, religion, age, disability, or any other protected class.
- Sustained derailing of technical discussions into ideological ones. There are better forums for those debates.
- Publishing private information of another contributor (real-life identity, workplace, location) without their consent.

## Reporting

If something happens that violates this code, email the project author (see the GitHub profile). All reports are kept confidential. The author is the sole arbiter during the hackathon judging window; after public release we'll move to a community process.

## Enforcement

First incident: private warning + ask for an apology to the affected party.
Second incident: temporary ban from issues / PRs / discussions.
Egregious or repeated: permanent ban.

The author reserves the right to skip steps for clearly bad-faith behavior (spam, harassment, doxing).

## Inspiration

This is shorter than the standard Contributor Covenant on purpose. The goal isn't to enumerate every possible violation — it's to set a clear bar: **honest, kind, scored**. If your contribution clears all three, you're welcome here.
